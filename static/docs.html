API 文档

面向浏览器会话与 AI Agent 的认证 HTTP API。错误统一为 {"error":{"code":"…","message":"…"}}。

/api/* REST POST /mcp MCP Cookie: vwa_session

01

认证与源站

  • 浏览器会话使用 HttpOnly、SameSite=Strict Cookie vwa_session，勿记录明文。
  • 非安全方法（POST/PUT/DELETE…）须带 Origin，且 host 与请求 Host 完全一致（same-origin）。
  • MCP 使用 Authorization: Bearer <VWA_MCP_TOKEN>，在有效 Bearer 下豁免同源检查。
  • GET /api/status 公开，不会加载模型。

02

REST 端点

方法 路径 认证 说明
GET /api/status 公开 服务状态：configured、model_present、model_ready、funclip_ready、mcp 等。
POST /api/setup 一次性 JSON {token, password}；password ≥ 12 字符。完成后删除 setup-token。
POST /api/auth/login 公开 JSON {password}；设置会话 Cookie。
POST /api/auth/logout 会话 清除会话。
GET /api/speakers 会话 列出说话人与参考音色档案。
POST /api/speakers 会话 JSON {name}。
PATCH /api/speakers/{id} 会话 JSON {name}，1–100 字符；名称冲突返回 409。
DELETE /api/speakers/{id} 会话 须先删除其全部 profiles。
POST /api/speakers/{id}/profiles 会话 multipart：audio、style_name、prompt_text、consent=true。参考音 5–30 秒。
PATCH /api/profiles/{id} 会话 JSON {style_name}，同一说话人下唯一。
DELETE /api/profiles/{id} 会话 删除未占用的参考档案。
POST /api/generations 会话 JSON {speaker_id, profile_id, target_text, speed}；speed 0.75–1.25；文案 ≤ 1200 字符。
GET /api/generations/{id}/audio 会话 返回 WAV 音频。
POST /api/videos/subtitles 会话 JSON {video_path} 相对于 VWA_VIDEO_INPUT_DIR；返回 {segments, srt}。

03

HTTP MCP

端点 POST /mcp。JSON-RPC 2.0 方法： initialize、notifications/initialized、 tools/list、tools/call。

工具 参数 说明
get_status — 服务 / 模型 / FunClip 状态
list_speakers — 说话人与 profiles
create_speaker {name} 创建说话人
delete_speaker {speaker_id} 删除说话人
rename_speaker {speaker_id, name} 重命名说话人
add_voice_profile {speaker_id, style_name, prompt_text, audio_path, confirm_rights} audio_path 须在 VWA_REFERENCE_INPUT_DIR 下；confirm_rights 必须为 true
delete_voice_profile {profile_id} 删除 profile
rename_voice_profile {profile_id, style_name} 重命名语气 / style
generate_speech {speaker_id, profile_id, target_text, speed?} 返回本地 audio_path
get_generation {generation_id} 查询生成任务
extract_video_subtitles {video_path} 路径须在 VWA_VIDEO_INPUT_DIR 下

集成时不得绕过同意确认、认证、文件大小/时长、逐字稿、所有权、路径沙箱或同源检查。

04

示例

状态

curl -sS "http://127.0.0.1:7860/api/status"

登录（浏览器同源）

curl -sS -c cookies.txt -b cookies.txt \
  -X POST "http://127.0.0.1:7860/api/auth/login" \
  -H "Origin: http://127.0.0.1:7860" \
  -H "Content-Type: application/json" \
  -d '{"password":"YOUR_PASSWORD"}'

MCP tools/list

curl -sS -X POST "http://127.0.0.1:7860/mcp" \
  -H "Authorization: Bearer $VWA_MCP_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'

字幕提取

curl -sS -c cookies.txt -b cookies.txt \
  -X POST "http://127.0.0.1:7860/api/videos/subtitles" \
  -H "Origin: http://127.0.0.1:7860" \
  -H "Content-Type: application/json" \
  -d '{"video_path":"clip.mp4"}'
